Detail User Stories per Role

Sistem menggunakan hierarki L1-L3 untuk memastikan operasional berjalan sesuai tanggung jawab.

Aktivasi Lisensi

Sebagai Owner, saya ingin menginput kode lisensi dari email agar aplikasi aktif permanen di perangkat ini (Fingerprint).

Manajemen Karyawan

Sebagai Owner, saya ingin membuat akun kasir/supervisor dengan PIN 6 digit tanpa batasan jumlah pengguna.

Impor Produk Massal

Sebagai Owner, saya ingin mengimpor ratusan produk dari Excel agar tidak perlu input manual satu per satu.

Operasional Kasir

Sebagai Owner, saya ingin bisa melayani transaksi kasir jika personil sedang kurang (Hierarki tertinggi).

Stock Opname

Sebagai Supervisor, saya ingin menyesuaikan stok fisik dengan sistem agar data inventaris selalu akurat.

Otorisasi Void

Sebagai Supervisor, saya ingin memberikan PIN otorisasi saat kasir perlu menghapus transaksi (void) demi keamanan.

Manajemen Barang Masuk

Sebagai Supervisor, saya ingin mencatat penambahan stok dari suplier secara lokal.

Buka/Tutup Shift

Sebagai Kasir, saya ingin memasukkan modal awal dan mencetak laporan shift saat closing (Serah terima).

Pencatatan Status Bayar

Sebagai Kasir, saya ingin memilih status (Tunai, QRIS, Debit) setelah pelanggan bayar via EDC eksternal.

Cetak Struk Bluetooth

Sebagai Kasir, saya ingin mencetak struk ESC/POS segera setelah transaksi sukses sebagai bukti pelanggan.

Laporan Shift Mandiri

Sebagai Kasir, saya ingin melihat ringkasan penjualan saya hari ini untuk memastikan uang di laci sesuai.

Simulasi Sinkronisasi Hybrid

Lihat bagaimana data tetap aman mengalir antar perangkat meski internet tidak stabil.

Dokumen PRD Lengkap (12 Poin)

Konsolidasi seluruh kebutuhan teknis dan fungsional.

01 Ringkasan Eksekutif

Sistem POS khusus UMKM dengan model Lifetime License (Lite) dan Subscription (Pro). Berjalan 100% offline menggunakan SQLite dengan aktivasi satu kali di awal via Backend Golang.

02 Target Pengguna

UMKM Ritel & F&B outlet tunggal yang ingin kepastian biaya, bisnis di area internet tidak stabil, dan pengusaha outlet tunggal dengan banyak karyawan.

03 Struktur Tier & Model Bisnis

Lite: Lifetime Pay, 1 Outlet, Offline Only. Pro: Subscription, Multi-outlet, Cloud Sync.

04 Fitur Utama (Fungsional)

  • Aktivasi Lisensi 1x (Online).
  • Hierarki Role (L1 Owner, L2 Supervisor, L3 Kasir).
  • Manajemen Stok & Produk (Manual/Excel).
  • Pencatatan Status Bayar (Tunai/QRIS/Debit).
  • Print ESC/POS Engine.

05 Aturan Validasi Data

Entitas Validasi
ProdukNama (3-100 char), SKU (Unique), Harga (>0)
UserPIN (6 Digit Numeric), Nama (Max 50 char)
NotaFormat: POS-[TGL]-[UNIK]

06 Spesifikasi Teknis & Stack

Backend: Go (Fiber). Database: SQLite (Local) & Supabase (License Master). Mobile: Flutter. Monitoring: Sentry.

07 User Stories (Detailed)

Tercakup dalam explorer interaktif di atas (Owner, Supervisor, Kasir).

08 Non-Functional Requirements (NFR)

Response < 200ms

Sync 99.9% Accuracy

09 Batasan Ruang Lingkup (Out of Scope)

Tidak ada integrasi otomatis payment gateway, Grab/GoFood, atau sinkronisasi multi-device di Tier 1.

10 Spesifikasi Hardware Min.

Android 10+, RAM 3GB+, Storage 1GB+, Bluetooth Thermal Printer.

11 Asumsi & Ketergantungan

Pengguna memiliki EDC/QRIS sendiri, sistem hanya pencatatan. Backend Go stabil untuk aktivasi awal.

12 Kriteria Kesuksesan (KPI)

Nol data korup saat offline, checkout < 2 detik, durasi onboarding < 15 menit.
