POSify PRD Viewer

Version 1.9 (Final MVP)

Ringkasan Eksekutif & Strategi

Aplikasi POSify dirancang khusus untuk UMKM dengan pendekatan Offline-First. Fokus utama rilis v1.9 ini adalah menonjolkan model bisnis sekali bayar (Tier 1) yang diiringi dengan peningkatan keamanan via AES-256 Encryption dan verifikasi Heartbeat.

🎯 Target Pengguna Utama

🏪

UMKM Ritel & F&B Kecil

Pengusaha yang menginginkan kepastian biaya operasional tanpa beban biaya langganan bulanan SaaS konvensional.

📶

Area Internet Tidak Stabil

Membutuhkan kecepatan operasional (< 1 detik) tanpa bergantung pada latensi cloud atau koneksi jaringan.

👥

Manajemen Karyawan Masif

Outlet tunggal dengan banyak staf (Kasir, SPV) yang butuh pembagian peran tanpa biaya tambahan per lisensi pengguna.

⚖️ Perbandingan Arsitektur Tier

Fokus MVP (v1.9)

Tier 1: Lite

Sekali Bayar (Lifetime License)

  • ✔️ 1 Outlet Terkunci: Lisensi terikat pada Device Fingerprint.
  • ✔️ Karyawan Tak Terbatas: Akun dikelola penuh via database lokal.
  • ✔️ Offline Murni: Hanya butuh internet 1x per 7 hari (Heartbeat).
  • ✔️ Penyimpanan: SQLite lokal dengan Drift ORM (Type-Safe).
  • ✔️ Keamanan: Auto-Backup dengan Enkripsi AES-256.
Roadmap Fase 4

Tier 2: Pro

Langganan (Subscription)

  • 🚀 Multi-Outlet: Ekspansi hingga 3 outlet terhubung.
  • 🚀 Cloud Sync: Sinkronisasi antar perangkat via PowerSync.
  • 🚀 Multi-Gudang: Manajemen inventaris terpusat.
  • 🚀 Penyimpanan Hybrid: SQLite lokal + Supabase Master.
  • 🚀 Otomasi: Cloud backup otomatis & Dashboard Web.

Hierarki Peran & Arsitektur Teknis

Sistem POSify menggunakan Otorisasi Kumulatif. Artinya, level yang lebih tinggi mewarisi semua kemampuan level di bawahnya. Klik pada profil di bawah ini untuk melihat distribusi kapasitas operasionalnya.

Kapasitas Owner

🔒

Sistem Keamanan PIN: Jika terjadi 5x kegagalan input PIN berturut-turut pada perangkat, akun akan diblokir sementara (*cooldown*) untuk mencegah peretasan.

Arsitektur Teknis & Dependensi

🟦

Frontend App

Dibuat dengan Flutter. Menghasilkan file Android APK. Mendukung UI reaktif berbasis state lokal.

🗄️

Database Lokal

SQLite via Drift ORM. Menjamin integritas data (*Type-Safety*) dan dapat berjalan 100% offline murni.

🌐

License Backend

Server mikro menggunakan Go (Fiber). Hanya bertugas memvalidasi *Device Fingerprint* dan memonitor *Heartbeat 24j*.

🔌

Hardware Integration

Mendukung protokol ESC/POS untuk Printer Thermal Bluetooth & sistem Continuous Scanning Barcode USB/BT.

Pusat Simulasi Alur Kerja (v1.9)

Pilih skenario operasional dari daftar di bawah. Fitur ini merepresentasikan perjalanan sistem (*System Workflow*) untuk fungsionalitas kunci, membantu Anda memahami interaksi antara pengguna, aplikasi, dan database.
